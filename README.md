## 🌐 이 저장소는 그대로 웹사이트가 됩니다

Day 5에 GitHub Pages를 켜는 순간, 저장소의 **모든 파일이 각자 인터넷 주소(URL)를 갖습니다.**

| 저장소 안 위치 | 역할 |
|------|------|
| `index.html` (루트) | 🏛 **대문(전시장)** — 자동으로 홈페이지가 됩니다 |
| `images/` | 🖼 **웹용 대표 이미지 5장** — 대문에 전시할 액자 |
| `day1-me/` ~ `day4-brand/` | 📦 **원본 자료실(수장고)** — 대문의 "산출물 보기" 링크로 열립니다 |

**그래서 매일의 커밋이 곧 웹사이트 콘텐츠입니다!** 지금 루트에 있는 index.html은 예시 — Day 5에 ChatGPT와 함께 나만의 디자인으로 교체하세요.

## 📤 파일 올리는 법 (매일 8교시 "오늘의 커밋" 시간)

1. 올릴 폴더 클릭 (예: `day1-me`)
2. `Add file` → **Upload files**
3. 파일을 끌어다 놓기 (여러 개 한 번에 OK)
4. 아래 커밋 메시지 쓰고 → 초록색 **Commit changes** 버튼 = 🎉 봉인 완료!

### ✍️ 커밋 메시지 예시

```
Day1: 나 사용설명서 포스터, 미래 명함 업로드
```

## 📛 파일 이름 규칙

**`day번호_내용_닉네임`** 형식으로! (한글 OK, 띄어쓰기 대신 밑줄 `_`)

| 좋은 예 ✅ | 나쁜 예 ❌ |
|-----------|-----------|
| `day1_나사용설명서_미래.png` | `최종본진짜최종.png` |
| `day2_인포그래픽_수진.png` | `제목없음(3).png` |
| `day4_룩북_하니.pdf` | `KakaoTalk_20260722.jpg` |

> ⚠️ **예외 — `images/` 폴더만은 영문!** 웹 주소에서 한글 파일명은 깨질 수 있어서, 웹사이트에 표시할 대표 이미지 5장은 **영문 소문자+밑줄**로 저장합니다:
> `profile.png` · `day1_poster.png` · `day2_infographic.png` · `day3_character.png` · `day4_brand.png`
