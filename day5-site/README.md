오늘의 미션: **3종 검수를 통과하고, 내 이름의 웹사이트를 세상에 연다!**

> ⚠️ `index.html`과 `images` 폴더는 **이 폴더가 아니라 저장소 맨 위(루트)** 로! (아래 참고)

## 🗂 배포 원리 — "저장소가 곧 웹사이트" (4교시)

GitHub Pages를 켜면 저장소의 **모든 파일이 각자 인터넷 주소(URL)를 갖습니다.**
오늘 새로 만드는 것은 딱 두 가지:

1. **`images/` 웹용 대표 이미지 5장** — 영문 파일명! ([images 폴더 안내](../images/) 참고)
   `profile.png` · `day1_poster.png` · `day2_infographic.png` · `day3_character.png` · `day4_brand.png`
2. **`index.html` 대문** — ChatGPT에게 요청 (이미지·폴더 경로를 프롬프트에 그대로 알려주는 게 핵심!)

나머지 `day1-me/`~`day4-brand/`는 4일간 커밋한 그대로 — 대문의 "산출물 보기" 링크로 열리는 원본 자료실이 됩니다.

## 🖥 만들기 (4~5교시) — 내 컴퓨터 작업 폴더에서

1. 바탕화면에 `my-portfolio-web` 폴더 → 그 안에 `images` 폴더 → 대표 이미지 5장을 **영문 파일명**으로 저장
2. ChatGPT가 만든 코드를 `index.html`로 저장 — 반드시 **`images` 폴더 옆**에!
3. `index.html` 더블클릭 → 미리보기에서 **이미지까지 보이면 성공** (안 보이면 경로·파일명부터! 배포해도 똑같이 안 보입니다)
4. 경로 규칙: **상대 경로만** 사용, 절대 슬래시(`/`)로 시작하지 않기
5. 💡 "산출물 보기" 링크는 내 컴퓨터에서는 안 열리는 게 정상 — **배포 후에 열립니다**

## 🚀 배포하기 (6교시)

1. 저장소 첫 화면 `Add file` → `Upload files` → **`index.html` 파일과 `images` 폴더를 통째로** 드래그 (⚠️ 반드시 루트에!)
   - 업로드 후 첫 화면에 `index.html` + `images/` + `day1-me/`~가 **나란히** 보이면 성공
2. 저장소 `Settings` → 왼쪽 메뉴 `Pages` → Branch `main` / 폴더 `/ (root)` → `Save`
3. 1~3분 기다린 후 `https://내아이디.github.io/저장소이름/` 접속
4. 열리면 🎉 **타임캡슐 개봉 성공!**
5. **연결 확인 투어**: ① 프로필 + 카드 이미지 4장이 다 보이나 ② "산출물 보기" 클릭 → 4일간 커밋한 원본 폴더가 열리나

## 🔍 배포 전 3종 검수 (필수 관문!)

- [ ] **팩트체크** — 기획서·인포그래픽의 숫자와 출처 확인했나요?
- [ ] **저작권** — 이미지·폰트 이용 범위 문제없나요?
- [ ] **개인정보** — 실명·연락처·학교 정보·얼굴 사진이 어디에도 없나요? (웹사이트는 전 세계 공개!)

## 🚨 안 될 때

| 증상 | 확인할 것 |
|------|-----------|
| 사이트가 안 열림 | 파일명이 정확히 `index.html`? 루트에 있나? Pages 설정 저장했나? 3분 더 기다렸나? |
| 사이트는 열리는데 이미지가 X | `images` 폴더가 루트에 올라갔나? 파일명 **대소문자까지** 일치? 경로가 `/`로 시작하지 않나? |
| "산출물 보기"가 404 | 코드 속 링크(`day1-me/` 등)와 실제 폴더명이 정확히 같은가? (오타·대소문자) |
